Repaso rápido

🧰Las tres herramientas base

Antes de arrancar el Cajero necesitamos tres cosas: leer datos del usuario, combinar condiciones y convertir tipos. Sin ellas, el resto no se sostiene.

1Scanner — leer datos del usuario

Definición

Scanner es una clase de Java que lee lo que el usuario escribe por teclado.

📝
En la vida real
Es como un formulario: el programa pregunta, el usuario responde, y esa respuesta queda guardada en una variable.

Para usar Scanner hay que importarlo arriba del archivo y crearlo dentro del main.

📁 Crea Repaso.java en la raíz del proyecto · córrelo con el botón ▶ Run (o F5)
import java.util.Scanner;

public class Repaso {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);

        System.out.print("Nombre del cliente: ");
        String nombre = sc.nextLine();

        System.out.print("Cuantas personas en la mesa? ");
        int personas = sc.nextInt();

        System.out.println("Mesa de " + nombre + ", " + personas + " personas.");
    }
}
🤔¿Por qué para el nombre uso nextLine() y para las personas nextInt()?
Cada uno lee un tipo distinto: nextLine() lee texto (String), nextInt() lee un entero, nextDouble() lee un decimal. El método debe coincidir con el tipo que esperas recibir.
⚠️
Error clásico que te va a perseguir: si lees primero con nextInt() y luego con nextLine(), el texto sale en blanco. Al leer el número, nextInt() deja el Enter pendiente y el nextLine() se lo "come". Solución: pon un sc.nextLine() extra después del nextInt() para limpiar ese Enter.

2Operadores lógicos — combinar condiciones

🎟️
Caso real
El descuento aplica si la cuenta supera 100.000 Y el cliente tiene cupón. No basta con cumplir una sola.
OperadorNombreDa true cuando…
&&AND (y)ambas condiciones son verdaderas
||OR (o)al menos una es verdadera
!NOT (no)invierte el valor
📁 En Repaso.java, dentro del main
double subtotal = 120000;
boolean tieneCupon = true;

boolean aplicaDescuento = subtotal > 100000 && tieneCupon;
System.out.println("Aplica descuento? " + aplicaDescuento);
🌍
Cada uno en la vida real
&& → para conducir necesitas licencia Y carro.  || → pagas con efectivo O tarjeta.  ! → "no está disponible": invierte el estado.
🤔Si tieneCupon fuera false, ¿qué imprime y por qué?
Imprime false. Aunque el subtotal supere 100.000, el && exige que las dos condiciones se cumplan.

3Conversión de tipos

💰
Caso real
Si divides la cuenta entre los comensales y guardas el resultado en un int, los centavos se pierden: un int no guarda decimales, los recorta.
Definición

Implícita (automática): de un tipo pequeño a uno grande. Java lo hace solo y no se pierde nada. Ej: int → double.

Explícita (casting): de un tipo grande a uno pequeño. La fuerzas tú con (tipo) y puede perder información. Ej: double → int.

📁 En Repaso.java
int unidades = 5;
double comoDecimal = unidades;
System.out.println("Implicita: " + comoDecimal);   // 5.0

double precio = 19990.75;
int precioRecortado = (int) precio;
System.out.println("Casting: " + precioRecortado);  // 19990
🥤
Analogía
Implícita = pasar agua de un vaso pequeño a uno grande: cabe sin problema. Explícita = del grande al pequeño: hay que forzarlo y se derrama (se pierde) lo que no cabe.

Bloque 1

🔀Que el programa decida

El Cajero debe aplicar descuento solo si la cuenta supera cierto monto. ¿Cómo sabe cuándo sí y cuándo no? Evaluando una condición y tomando caminos distintos según el resultado.

Definición

Un condicional ejecuta un bloque de código solo si una condición es verdadera (true).

☔
En la vida real
"Si llueve, entonces llevo paraguas; si no, salgo sin él."

Ej. 1if simple

📁 Crea Decisiones.java en la raíz
public class Decisiones {
    public static void main(String[] args) {
        double cuenta = 150000;

        if (cuenta > 100000) {
            System.out.println("Aplica descuento");
        }
    }
}

Esto solo cubre un caso. ¿Y si no aplica? Falta el otro camino.

Ej. 2if / else if / else

Cubrimos todos los casos, clasificando la cuenta por monto:

📁 En Decisiones.java (reemplaza el main)
double cuenta = 150000;

if (cuenta >= 200000) {
    System.out.println("Descuento VIP (15%)");
} else if (cuenta > 100000) {
    System.out.println("Descuento normal (10%)");
} else {
    System.out.println("Sin descuento");
}
🤔Si la cuenta es 250.000, ¿por qué no imprime también "Descuento normal"? (250.000 también es > 100.000)
Imprime solo "Descuento VIP". Las condiciones se evalúan en orden y, en cuanto una se cumple, Java ejecuta esa y se sale de toda la cadena. No revisa las demás.
🚫
Error típico: comparar con = en vez de ==, como if (cuenta = 100000). No compila: = asigna, == compara. Java espera una condición, no una asignación. El error se lee como "incompatible types: ... cannot be converted to boolean".
⚠️
Cuidado con dos detalles: poner ; justo después del if (...) "vacía" el if y el bloque corre siempre. Y el orden importa: si pones la condición más amplia primero, las específicas nunca se alcanzan.

Bloque 2

📋Muchas opciones exactas: switch

El Cajero tiene un menú: 1 nuevo pedido, 2 aplicar descuento, 3 cerrar caja. Podrías escribir un else if por cada número, pero cuando comparas un mismo valor contra varias opciones exactas, el switch es más legible.

Definición

switch evalúa una variable y salta directo al case que coincide.

🏧
En la vida real
El menú del cajero automático: marcas una opción y ejecuta esa, no las demás.

Ej. 1switch con int (el menú)

📁 Crea Opciones.java en la raíz
public class Opciones {
    public static void main(String[] args) {
        int opcion = 2;

        switch (opcion) {
            case 1:
                System.out.println("Nuevo pedido");
                break;
            case 2:
                System.out.println("Aplicar descuento");
                break;
            case 3:
                System.out.println("Cerrar caja");
                break;
            default:
                System.out.println("Opcion invalida");
        }
    }
}
🚫
Si quitas los break: ocurre el fall-through. Al entrar a un case, sigue ejecutando los siguientes hasta hallar un break o terminar. En esta forma clásica el break es obligatorio.

🔎 ¿El switch solo recibe números?

🤔¿Un switch solo sirve con int, o puede comparar también texto?
También sirve con texto y más. Según la documentación oficial de Oracle, un switch acepta: enteros byte, short, char, int (y sus envoltorios Byte, Short, Character, Integer), String (desde Java 7) y tipos enum (desde Java 5).

No acepta: boolean, long, float ni double.

Ej. 2switch con String (por nombre)

📁 En Opciones.java (reemplaza el main)
String comando = "descuento";

switch (comando) {
    case "pedido":
        System.out.println("Nuevo pedido");
        break;
    case "descuento":
        System.out.println("Aplicar descuento");
        break;
    case "cerrar":
        System.out.println("Cerrar caja");
        break;
    default:
        System.out.println("Comando no reconocido");
}
⚠️
Si la variable String llega en null, el switch lanza NullPointerException. Asegúrate de que traiga un valor.
🌟
Java moderno: desde Java 14 existe la forma con flecha -> que no necesita break y evita el fall-through. Primero domina la forma clásica para entender el flujo; en proyectos reales se prefiere la flecha porque elimina el error del break olvidado.
switch (opcion) {
    case 1 -> System.out.println("Nuevo pedido");
    case 2 -> System.out.println("Aplicar descuento");
    case 3 -> System.out.println("Cerrar caja");
    default -> System.out.println("Opcion invalida");
}
💡
switch compara valores exactos, no rangos como >= 90 — para eso usa if / else if. El default no es obligatorio, pero es buena práctica para cubrir lo inesperado.

Bloque 3

🔁Repetir sin copiar: ciclos

El Cajero atiende muchos pedidos en un turno. ¿Lanzas el programa de nuevo por cada uno? No: cuando una acción se repite, usamos un ciclo — escribimos la lógica una vez y le decimos cuántas (o hasta cuándo) correr.

Definición

Un ciclo ejecuta un bloque varias veces mientras se cumpla una condición.

Mismo objetivo —repetir— pero cada uno encaja en un escenario distinto.

forcuando sí sé cuántas veces

🤔¿Qué necesita un ciclo para no repetirse infinitamente?
Tres partes: inicio (desde dónde), condición (hasta cuándo) y avance (cómo cambia). Sin avance, nunca termina.
📁 Crea Ciclos.java en la raíz
public class Ciclos {
    public static void main(String[] args) {
        for (int i = 1; i <= 5; i++) {
            System.out.println("Plato #" + i + " impreso en el recibo");
        }
    }
}

Otro for, esta vez sumando un acumulado:

int totalPlatos = 0;
for (int i = 1; i <= 4; i++) {
    totalPlatos = totalPlatos + 1;
}
System.out.println("Total de platos: " + totalPlatos);
💡
En (int i = 1; i <= 5; i++): int i = 1 es el inicio, i <= 5 es la condición, i++ es el avance (suma 1 cada vuelta).

whilecuando sé la condición de parada

int mesasPorLimpiar = 3;
while (mesasPorLimpiar > 0) {
    System.out.println("Faltan " + mesasPorLimpiar + " mesas");
    mesasPorLimpiar--;
}
System.out.println("Todo limpio!");

Otro ejemplo: vaciar una fila de clientes.

int clientesEnFila = 5;
while (clientesEnFila > 0) {
    System.out.println("Atendiendo cliente, quedan " + (clientesEnFila - 1));
    clientesEnFila--;
}

do whileal menos una vez sí o sí

📋
Caso real
El menú del Cajero debe mostrarse al menos una vez, aunque el usuario quiera cerrar de inmediato. El do while ejecuta primero y revisa la condición después.
📁 En Ciclos.java (reemplaza el main · necesita el import de Scanner)
import java.util.Scanner;

public class Ciclos {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        int opcion;

        do {
            System.out.println("1. Continuar   2. Salir");
            System.out.print("Elige: ");
            opcion = sc.nextInt();
        } while (opcion != 2);

        System.out.println("Saliste del menu");
    }
}

Otro uso muy común: pedir un dato válido sí o sí.

Scanner sc = new Scanner(System.in);
int edad;

do {
    System.out.print("Edad del cliente (mayor a 0): ");
    edad = sc.nextInt();
} while (edad <= 0);

System.out.println("Edad valida: " + edad);
🤔¿En qué se diferencia el do while del while normal?
El while revisa la condición antes (puede no ejecutarse nunca). El do while ejecuta primero y revisa después, así que siempre corre al menos una vez.
♾️
Ciclo infinito: si en el while olvidas el mesasPorLimpiar--, la condición nunca cambia y el programa repite para siempre (se corta con Ctrl + C). Regla de oro: dentro del ciclo siempre debe cambiar algo que afecte la condición.

Bloque 4

🧮Reutilizar lógica: métodos

El cálculo de propina y total lo necesitas en varias partes del Cajero. ¿Copias la fórmula en cada lugar? Nunca: si cambia, tendrías que corregir varios sitios. La lógica se escribe una vez y se reutiliza llamándola. Eso es un método.

Definición

Un método es un bloque de código con nombre que hace una tarea y se puede invocar cuantas veces quieras.

🍳
En la vida real
Una receta: la escribes una vez y la sigues siempre que la necesites. Los ingredientes son los parámetros (datos que entran); el plato es el retorno (resultado que sale).

Ej. 1método que calcula la propina

📁 Crea Metodos.java · el método va dentro de la clase pero fuera del main
public class Metodos {
    public static void main(String[] args) {
        double propina = calcularPropina(80000, 0.10);
        System.out.println("Propina: " + propina);
    }

    static double calcularPropina(double subtotal, double porcentaje) {
        return subtotal * porcentaje;
    }
}
🤔¿Por qué va double antes del nombre del método?
Es el tipo de dato que el método devuelve. Si fuera void, no devolvería nada. Como entrega un decimal, devuelve double.

Ej. 2método que devuelve el total con propina

📁 En Metodos.java (agrega otro método)
static double calcularTotal(double subtotal, double porcentaje) {
    double propina = calcularPropina(subtotal, porcentaje);
    return subtotal + propina;
}
💡
Fíjate: calcularTotal reutiliza calcularPropina. Un método puede llamar a otro.

🧠 Scope (alcance de variables)

🤔La variable propina que está dentro de calcularTotal, ¿puedo usarla en el main?
No. propina solo existe dentro de su método; al terminar, desaparece. Eso es el scope: una variable vive únicamente dentro del bloque { } donde nació. La única forma de sacar un dato del método es el return.
🚪
Analogía
Lo que se habla en una sala no existe en la otra. Cada bloque tiene su propio espacio.
🚫
Si intentas imprimir propina desde el main, Java marca "cannot find symbol": no existe en ese espacio. Y si olvidas el return en un método que promete devolver algo → "missing return statement".
❓
¿Qué es static? Por ahora: permite llamar el método directo desde main sin crear un objeto. El porqué profundo llega en la Clase 5 (POO).
Así va la app 🧾 El Cajero ya calcula

Calcula el total con un método reutilizable.

📁 Cajero.java
import java.util.Scanner;

public class Cajero {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        int opcion;

        do {
            System.out.println("\n--- CAJERO ---");
            System.out.println("1. Nuevo pedido");
            System.out.println("2. Cerrar caja");
            System.out.print("Elige una opcion: ");
            opcion = sc.nextInt();

            switch (opcion) {
                case 1:
                    System.out.print("Subtotal: ");
                    double subtotal = sc.nextDouble();
                    double total = calcularTotal(subtotal, 0.10);
                    System.out.println("Total con propina: " + total);
                    break;
                case 2:
                    System.out.println("Caja cerrada");
                    break;
                default:
                    System.out.println("Opcion invalida");
            }
        } while (opcion != 2);
    }

    static double calcularTotal(double subtotal, double porcentaje) {
        double propina = subtotal * porcentaje;
        return subtotal + propina;
    }
}
Bloque 5

🧩Organización profesional

En un restaurante el chef no cobra las cuentas ni el cajero cocina: cada uno tiene una responsabilidad clara. El código profesional es igual — una clase, una responsabilidad. Eso es separación de responsabilidades, base del principio SRP de SOLID que verás formalmente más adelante.

El Cajero no sabe cómo se calcula — solo a quién pedírselo.

La clase de lógica

📁 Crea Pedido.java en la raíz (misma carpeta que Cajero.java)
public class Pedido {
    double calcularTotal(double subtotal, double porcentaje) {
        double propina = subtotal * porcentaje;
        return subtotal + propina;
    }
}

La clase principal — versión final

📁 Cajero.java queda solo con menú y coordinación
import java.util.Scanner;

public class Cajero {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        Pedido pedido = new Pedido();
        int opcion;

        do {
            System.out.println("\n--- CAJERO ---");
            System.out.println("1. Nuevo pedido");
            System.out.println("2. Cerrar caja");
            System.out.print("Elige una opcion: ");
            opcion = sc.nextInt();

            switch (opcion) {
                case 1:
                    System.out.print("Subtotal: ");
                    double subtotal = sc.nextDouble();
                    double total = pedido.calcularTotal(subtotal, 0.10);
                    System.out.println("Total con propina: " + total);
                    break;
                case 2:
                    System.out.println("Caja cerrada");
                    break;
                default:
                    System.out.println("Opcion invalida");
            }
        } while (opcion != 2);
    }
}
🤔¿Por qué Cajero puede usar calcularTotal si está en otro archivo?
Porque creó un objeto Pedido con new Pedido() y a través de él pide el cálculo. Cajero no sabe cómo se calcula —y no le importa—; solo sabe a quién pedírselo. Eso es separar responsabilidades.
▶️

📌
Ambos .java van en la raíz, sin carpetas. Todavía no usamos package; eso llega más adelante. ¿Esto ya es POO? Es la antesala: en la Clase 5 le damos nombre completo (clases, objetos, atributos y encapsulación).
